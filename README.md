# KNRM (Kernel-based Neural Ranking Model) для определения дубликатов вопросов

## 📝 Описание проекта

Это самописная реализация модели ранжирования KNRM (Kernel-based Neural Ranking Model) для задачи определения дубликатов вопросов на основе датасета Quora Question Pairs (QQP). Модель обучается с использованием триплетного подхода (triplet loss) для обучения ранжированию релевантности между парами текстов.

### Особенности реализации:
- Полностью самописная архитектура на PyTorch
- Использование предобученных GloVe эмбеддингов
- Триплетное обучение для ранжирования
- Поддержка кастомных генераторов случайных чисел без зависимости от NumPy
- Валидация с метрикой NDCG@10

## 🏗️ Архитектура модели

### KNRM (Kernel-based Neural Ranking Model)
Модель использует ядерные функции для сравнения запроса и документа:
1. **Embedding layer** - преобразование токенов в векторные представления (GloVe)
2. **Interaction matrix** - вычисление попарных косинусных сходств между токенами запроса и документа
3. **Kernel pooling** - применение RBF-ядер для извлечения признаков
4. **MLP** - полносвязные слои для финальной оценки релевантности

### Особенности архитектуры:
- **Freeze embeddings** - возможность заморозки эмбеддингов GloVe
- **Настраиваемое количество ядер** - параметр `knrm_kernel_num`
- **Гибкая MLP-головка** - настраиваемые размеры выходных слоев

## 📊 Датасет

Проект использует датасет Quora Question Pairs (QQP) в формате:
- **id_left** - идентификатор первого вопроса (запрос)
- **id_right** - идентификатор второго вопроса (документ)
- **text_left** - текст первого вопроса
- **text_right** - текст второго вопроса  
- **label** - метка (1 - дубликат, 0 - не дубликат)
